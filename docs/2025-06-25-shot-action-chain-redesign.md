# 分镜动作链重设计 — 导演思维节奏控制

## 背景

当前分镜拆解输出的动作链描述过于机械，仅是剧本关键词的简单提取，缺乏导演的画面思维、情绪铺垫和节奏控制。

## 目标

1. 动作链描述体现导演的节奏设计意图
2. 时间拆分反映情绪弧线（铺垫→积累→爆发→收束）
3. 每条动作链包含镜头运动、情绪意图、节奏角色

## 方案：BC 结合

### 1. 分镜级别新增字段

```json
"rhythm_notes": "本分镜的节奏设计思路"
```

`rhythm_notes` 是导演对该分镜整体节奏的说明，包括：
- 情绪弧线的设计（如何从 A 状态到 B 状态）
- 各动作链的时间分配理由
- 镜头运动策略的总体思路

### 2. 动作链从三元组改为结构化对象

**之前（数组三元组）：**
```json
["0.0s-4.5s", "VCU", "描述"]
```

**之后（对象）：**
```json
{
  "time_range": "0.0s-3.5s",
  "shot_size": "VCU",
  "action": "含@标记的完整动作/画面描述（姿态、表情、位移、交互）",
  "camera_movement": "镜头运动方式英文术语（Push In / Slow Pan / Whip Pan / Track 等）",
  "emotional_intent": "此刻观众应感受到的情绪",
  "rhythm_role": "铺垫 | 积累 | 爆发 | 过渡 | 收束"
}
```

### 3. 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `time_range` | string | 时间区间，前一段结束时间即后一段起始时间，第一段从 0.0s 开始 |
| `shot_size` | string | 景别，英文术语（VCU/CU/MCU/MS/FS/VFS 等） |
| `action` | string | 详细动作/画面描述：角色姿态、表情变化、空间位移、@标记、文字标识、物体交互 |
| `camera_movement` | string | 镜头运动方式英文术语，无运动时写 Static |
| `emotional_intent` | string | 观众此刻应感受到的情绪说明 |
| `rhythm_role` | string | 节奏角色：该动作链在整个分镜节奏弧中的功能定位 |

### 4. 节奏角色定义

| 角色 | 典型时长占比 | 说明 |
|------|-------------|------|
| 铺垫 | 20-30% | 建立画面空间和情境，节奏舒缓 |
| 积累 | 25-35% | 情绪逐渐递进、紧张度上升 |
| 爆发 | 15-25% | 情绪/动作的高潮点，节奏最快 |
| 过渡 | 10-20% | 视角/场景平滑转换 |
| 收束 | 10-15% | 情绪释放后的沉淀，留白 |

### 5. 修改范围

| 文件 | 改动 |
|------|------|
| `extractors/shots.py` | DETAIL_SYSTEM: 新增节奏设计指引、修改 JSON 模板、修改核心规则 |
| `reports/word_report.py` | 适配新对象格式 |
| `reports/html_report.py` | 适配新对象格式 |
| `templates/report.html` | 适配新对象格式 |
| `static/js/app.js` | showResults 中适配新对象格式 |

### 6. 向下兼容

旧的数组格式 `["time", "shot_size", "action"]` 在 report 层做兼容处理：如果检测到数组而非对象，按旧格式渲染。确保已生成的报告不会报错。

### 7. 一条动作链最终输出示例

```
时间区间: 0.0s-3.5s
景别: VCU
动作: @小红从远处走来，身影逐渐在画面中央放大，@小明站在原地目光追随，嘴角开始微微上扬
镜头运动: Slow Push In（缓慢推近）
情绪意图: 营造久别重逢前的期待感，节奏舒缓，让观众和小明一起等待
节奏角色: 铺垫 — 情绪积累期
```
